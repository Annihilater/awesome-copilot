# 从存储库结构创建 LLMs.txt 文件

根据 https://llmstxt.org/ 上的官方 llms.txt 规范，在存储库的根目录中从头开始创建一个新的 `llms.txt` 文件。此文件为大型语言模型 (LLM) 提供高级指导，以查找相关内容以了解存储库的用途和规范。

## 主要指令

创建一个全面的 `llms.txt` 文件，作为 LLM 有效理解和导航存储库的入口点。该文件必须符合 llms.txt 规范，并针对 LLM 使用进行优化，同时保持人类可读性。

## 分析和规划阶段

在创建 `llms.txt` 文件之前，您必须完成彻底的分析：

### 第 1 步：审查 llms.txt 规范

- 审查 https://llmstxt.org/ 上的官方规范以确保完全合规
- 了解所需的格式结构和指南
- 注意特定的 markdown 结构要求

### 第 2 步：存储库结构分析

- 使用适当的工具检查完整的存储库结构
- 确定存储库的主要目的和范围
- 对所有重要目录及其用途进行分类
- 列出对 LLM 理解有价值的关键文件

### 第 3 步：内容发现

- 识别 README 文件及其位置
- 查找文档文件（`/docs/`、`/spec/` 等中的 `.md` 文件）
- 定位规范文件及其用途
- 发现配置文件及其相关性
- 查找示例文件和代码示例
- 识别任何现有的文档结构

### 第 4 步：创建实施计划

根据您的分析，创建一个结构化计划，其中包括：

- 存储库目的和范围摘要
- 对 LLM 理解至关重要的文件的优先级排序列表
- 提供附加上下文的辅助文件
- llms.txt 文件的组织结构

## 实施要求

### 格式合规性

`llms.txt` 文件必须遵循规范中的确切结构：

1. **H1 标题**：包含存储库/项目名称的单行（必需）
2. **块引用摘要**：块引用格式的简要说明（可选但推荐）
3. **其他详细信息**：零个或多个没有标题的 markdown 部分，用于提供上下文
4. **文件列表部分**：零个或多个包含 markdown 链接列表的 H2 部分

### 内容要求

#### 必需元素

- **项目名称**：清晰、描述性的标题，作为 H1
- **摘要**：解释存储库目的的简洁块引用
- **关键文件**：按类别组织的基本文件（H2 部分）

#### 文件链接格式

每个文件链接必须遵循：`[描述性名称](相对 url): 可选描述`

#### 部分组织

将文件组织到逻辑 H2 部分中，例如：

- **文档**：核心文档文件
- **规范**：技术规范和要求
- **示例**：示例代码和用法示例
- **配置**：设置和配置文件
- **可选**：辅助文件（特殊含义 - 可跳过以获得更短的上下文）

### 内容指南

#### 语言和风格

- 使用简洁、清晰、明确的语言
- 避免使用没有解释的行话
- 为人类和 LLM 读者编写
- 在描述中要具体和翔实

#### 文件选择标准

包括以下文件：

- 解释存储库的目的和范围
- 提供基本的技术文档
- 显示用法示例和模式
- 定义接口和规范
- 包含配置和设置说明
