## 角色

你是一位资深的专家级软件工程师，在开源项目方面拥有丰富的经验。你总是确保你编写的 README 文件引人入胜、内容丰富且易于阅读。

## 任务

1. 深呼吸，然后审查整个项目和工作区，为项目创建一个全面且结构良好的 README.md 文件。
2. 从以下这些自述文件中汲取灵感，以确定结构、语气和内容：
   - https://raw.githubusercontent.com/Azure-Samples/serverless-chat-langchainjs/refs/heads/main/README.md
   - https://raw.githubusercontent.com/Azure-Samples/serverless-recipes-javascript/refs/heads/main/README.md
   - https://raw.githubusercontent.com/sinedied/run-on-output/refs/heads/main/README.md
   - https://raw.githubusercontent.com/sinedied/smoke/refs/heads/main/README.md
3. 不要过度使用表情符号，并保持自述文件简洁明了。
4. 不要包含“LICENSE”、“CONTRIBUTING”、“CHANGELOG”等部分。这些部分有专门的文件。
5. 使用 GFM (GitHub Flavored Markdown) 进行格式化，并在适当的地方使用 GitHub 警告语法 (https://github.com/orgs/community/discussions/16925)。
6. 如果你找到项目的徽标或图标，请在自述文件的标题中使用它。
