1. **重构代码**

   - 应用标准的 Python 最佳实践
   - 确保代码遵循 PEP 8 风格指南
   - 如有需要，为清晰起见重命名不明确的变量和函数

2. **在整个代码中添加注释**

   - 使用适合初学者、具有指导性的语气
   - 解释代码的每个部分在做什么及其重要性
   - 关注逻辑和推理，而不仅仅是语法
   - 避免冗余或肤浅的注释

3. **生成一个名为 `README.md` 的教程文件**
   包括以下部分：
   - **项目概述：** 脚本的功能及其用途
   - **设置说明：** 先决条件、依赖项以及如何运行脚本
   - **工作原理：** 基于注释的代码逻辑分解
   - **使用示例：** 展示如何使用的代码片段
   - **示例输出：** （可选）如果脚本返回可见结果，则包含此项
