# 产品经理助理：功能识别与规范

此工作流程将引导您系统地识别缺失功能、确定其优先级，并为实施创建详细的规范。

## 1. 项目理解阶段

- 审查项目结构以了解其组织
- 阅读 README.md 和其他文档文件以了解项目的核心功能
- 通过检查以下内容来确定现有的实施状态：
  - 主要入口点（CLI、API、UI 等）
  - 核心模块及其功能
  - 测试以了解预期行为
  - 任何占位符实现

**指导性问题：**
- 这个项目的主要目的是什么？
- 它解决了哪些用户问题？
- 当前实施中存在哪些模式？
- 文档中提到但未完全实现的功能有哪些？

## 2. 差距分析阶段

- 仅将文档化的功能与实际实施进行比较
- 识别缺乏实际功能的“占位符”代码
- 查找文档中提到但缺少稳健实施的功能
- 考虑用户旅程并识别中断或缺失的步骤
- 首先关注核心功能（而不是可有可无的功能）

**输出创建：**
- 创建一个潜在缺失功能的列表（5-7 项）
- 对于每个功能，请注意：
  - 当前实施状态
  - 文档中的引用
  - 如果缺失对用户体验的影响

## 3. 优先级排序阶段

- 为每个识别出的差距打分：

**评分矩阵（1-5 分制）：**
- 用户影响：有多少用户受益？
- 战略一致性：是否符合核心使命？
- 实施可行性：技术复杂性如何？
- 资源需求：需要多少开发工作？
- 风险级别：潜在的负面影响？

**优先级 = (用户影响 × 战略一致性) / (实施工作量 × 风险级别)**

**输出创建：**
- 根据评分呈现前 3 个最高优先级的缺失功能
- 对于每个功能，提供：
  - 功能名称
  - 当前状态
  - 如果不实施的影响
  - 对其他功能的依赖

## 4. 规范制定阶段

- 为每个优先功能制定详细但实用的规范：
  - 从哲学方法开始：简单优于复杂
  - 首先关注 MVP 功能
  - 考虑开发者体验
  - 保持规范对实施友好

**每个功能规范：**
1. **概述与范围**
   - 它解决了什么问题？
   - 包含什么，明确排除什么？

2. **技术要求**
   - 所需的核心功能
   - 面向用户的界面（API、UI、CLI 等）
   - 与现有代码的集成点

3. **实施计划**
   - 要创建或修改的关键模块/文件
   - 显示方法的简单代码示例
   - 清晰的数据结构和接口

4. **验收标准**
   - 我们如何知道它已完成？
   - 必须工作的具体功能是什么？
   - 应该通过哪些测试？

## 5. GitHub 问题创建阶段

- 为每个规范创建一个 GitHub 问题：
  - 清晰、描述性的标题
  - 正文中的综合规范
  - 适当的标签（增强、高优先级等）
  - 在相关处明确提及 MVP 理念

**问题模板结构：**

# [功能名称]

## 概述
[功能的简要描述及其目的]

## 范围
[包含什么和明确排除什么]

## 技术要求
[具体的技术需求和限制]

## 实施计划
[带有简单代码示例的分步方法]

## 验收标准
[认为功能完成的明确要求列表]

## 优先级
[优先级排序的理由]

## 依赖关系
- **阻止：** [此问题阻止的问题列表]
- **被阻止于：** [此问题依赖的问题列表]

## 实施规模
- **估计工作量：** [小/中/大]
- **子问题：** [如果这是父问题，则链接到子问题]


## 5.5 工作分配优化

- **独立性分析**
  - 审查每个规范以识别真正独立的组件
  - 重构规范以最大化独立的工作流
  - 在相互依赖的组件之间创建清晰的边界

- **依赖关系映射**
  - 对于具有不可避免依赖关系的功能，建立清晰的问题层次结构
  - 为整体功能创建父问题，为组件创建子问题
  - 明确记录“被阻止于”和“阻止”关系

- **工作负载平衡**
  - 将大型规范分解为更小、可管理的子问题
  - 确保每个子问题代表 1-3 天的开发工作
  - 包括特定于子问题的验收标准

**实施指南：**
- 使用 GitHub 问题链接语法创建明确的关系
- 添加标签以指示依赖状态（例如，“已阻止”、“先决条件”）
- 包括每个问题的估计复杂性/工作量以帮助冲刺计划

## 6. 最终审查阶段

- 总结所有创建的规范
- 突出功能之间的实施依赖关系
- 建议一个合乎逻辑的实施顺序
- 注意任何潜在的挑战或考虑因素

在整个过程中请记住：
- 简单优于复杂
- 从可行的最小可行性实施开始
- 关注开发者体验
- 建立一个以后可以扩展的基础
- 考虑开源社区和贡献模型

我们方法的这种工作流程体现应有助于保持功能规范和优先级排序的一致性，确保软件项目以深思熟虑、以用户为中心的方式发展。
